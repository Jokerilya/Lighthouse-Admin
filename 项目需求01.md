技术栈:Vue3+Vite + TS + Pinia + Element Plus + UnoCSS+Mock.js

核心架构设计(当写到以下模块要特别注意):
1.Axios 二次封装：
统一处理 Loading 状态。
拦截器处理 Token 注入和 401 (登录过期) 跳转。

2.Mock.js 集成：
利用 vite-plugin-mock 插件，在 src/mock 下编写接口定义。
模拟：POST /api/login (返回 Token), GET /api/user/info (返回权限信息)。

3.UnoCSS 预设：
配置 preset-uno 和 preset-icons（直接用 i-ep-plus 这种图标，连图标组件都不用引了）。

4.Pinia 状态管理：
userStore：存储用户信息、Token、菜单权限。
appStore：控制侧边栏折叠、主题色、通知。

5.路由守卫：
动态生成路由：根据 userStore 中的菜单权限动态生成路由表。
路由拦截：未登录跳转登录页，已登录且访问白名单页面跳转首页。

功能模块
1. 登录与鉴权 (Authentication)
登录页：账号密码校验、验证码模拟。
路由守卫 (Permission Guard)：
未登录访问重定向到 /login。
登录后，根据 Mock 返回的 roles 动态挂载路由。
退出登录：清除 Pinia 状态和本地缓存。

2. 系统主框架 (Layout)
侧边栏 (Sidebar)：基于 el-menu 递归渲染，支持多级菜单和折叠切换。
顶部导航 (Header)：面包屑 (Breadcrumb)、全屏切换、用户头像下拉菜单。
多页签 (TagsView)：右键刷新、关闭当前/其他/全部页签（这是练习 Pinia 缓存的最佳场景）。

3. 用户与权限管理 (System Management) —— 核心 CRUD 练习
用户管理：
实现搜索、列表展示（el-table）、新增/编辑对话框（el-form）。
练习点：封装一个通用的 useTable Hook，把分页、查询、重置逻辑抽离出来。
角色/菜单权限：
实现一个树形控件（el-tree），用来勾选角色拥有的权限。

4. 个人管理插件 (Lighthouse 特色功能)
既然你的初衷是做个人管理，建议加两个特色模块：
任务清单 (Todo List)：
支持状态切换（已完成/未完成）、优先级标记。
练习 Drag & Drop（推荐使用 vue-draggable-plus）。
个人周报/记事本：
集成一个轻量级的 Markdown 编辑器（如 md-editor-v3）。
练习如何在 Vue3 中处理长文本和预览。

5. 仪表盘 (Dashboard)
数据大盘：
顶部展示 4 个统计卡片（如：任务完成数、学习时长等）。
中部展示一个折线图（练习 ECharts 结合 onMounted 与容器缩放适配）。
